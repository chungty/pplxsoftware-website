Questions your security team will ask

On Atlassian's infrastructure, in your selected data residency region. TeamOps uses Forge's built-in storage, which inherits Atlassian's data residency controls. PPLX Software has no servers — there is nowhere else for data to go.
No. Forge apps run in an isolated sandbox on Atlassian's infrastructure. The vendor (PPLX Software) cannot access, read, export, or view any tenant data. This is enforced at the platform level by Atlassian, not by our code.
TeamOps runs entirely on Atlassian's SOC 2 Type II and ISO 27001 certified platform. Because Forge apps execute within Atlassian's infrastructure, they inherit the platform's compliance certifications. There is no separate vendor infrastructure to audit.
Nothing changes for TeamOps. We built on Forge from day one — there is no migration path needed. Connect apps (which run on external vendor servers) are being deprecated by Atlassian in Q4 2026. Teams using Connect-based HR tools will need to find replacements.
Forge apps run inside Atlassian's sandbox with automatic security controls including network egress restrictions, scoped permissions, and data isolation.
